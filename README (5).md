# Syntecxhub_Amazon_Customers_Sentiment_Analysis

Sentiment analysis on Amazon product reviews — built for the Syntecxhub
internship Project 2 brief.

## What this project does
- Cleans and preprocesses raw review text (lowercasing, punctuation/URL
  removal, stopword filtering, tokenization).
- **Lexicon-based sentiment classification** (positive / negative / neutral)
  using a hand-built polarity word list — fast and fully unsupervised.
- **Supervised ML classifier** (TF-IDF + Logistic Regression) trained on
  rating-derived sentiment labels, evaluated with accuracy, a full
  classification report, and a confusion matrix.
- Identifies patterns in feedback: sentiment by star rating, by product,
  and over time.
- Generates 6 visualizations and a written insights report with concrete
  product-improvement recommendations.

## Using a real dataset
This repo ships with a synthetic dataset (`data/amazon_reviews.csv`,
generated by `src/generate_data.py`) so the pipeline runs out of the box.
To use a real dataset (e.g. the Kaggle "Amazon Product Reviews" dataset,
linked in the assignment slide), replace `data/amazon_reviews.csv` with a
CSV containing these columns and re-run `python src/main.py`:

| column | description |
|---|---|
| `review_id` | unique id |
| `product` | product name |
| `rating` | 1–5 star rating |
| `date` | review date (YYYY-MM-DD) |
| `review_text` | raw review text |

## Methodology notes
- **Labels**: Ratings ≥4 → positive, =3 → neutral, ≤2 → negative. This
  rating-derived label is used both as a sanity check for the lexicon
  scorer and as training/evaluation labels for the ML classifier.
- **Lexicon scorer**: counts occurrences of curated positive/negative
  words per review; net polarity determines the label. Easily extendable
  by adding words to `POSITIVE_WORDS` / `NEGATIVE_WORDS` in
  `sentiment_model.py`.
- **ML classifier**: TF-IDF (unigrams + bigrams, top 2000 features) feeding
  a balanced Logistic Regression model, with a stratified 75/25 train/test
  split.
